  A browser-based network engineering toolkit with IPv4/IPv6 subnetting, multicast, routing, switching, diagnostics, and more. Built with React and runs entirely client-side.

## Project Structure

| Path | Description |
|------|-------------|
| `rda-net-kit.html` | **Source** — loads React/Babel from CDN. Requires internet; serve via HTTP for dev. |
| `rda-net-kit-file.html` | **file:// build** — generated by `build-dev.js`; all JSX inlined, no XHR needed. (gitignored) |
| `rda-net-kit-offline.html` | **Offline build** — all JS, CSS and fonts inlined by `bundle.js`. Works anywhere. (gitignored) |
| `components/` | Individual JSX component files (one per tool). Edit these during development. |
| `ip-utils.js` | Standalone IPv4/IPv6 utility library (exports `window.IPv4` / `window.IPv6`). |
| `oui-db.js` | MAC address OUI vendor database (~39K entries). Sourced from [maclookup.app](https://maclookup.app/downloads/json-database) (free with attribution). |
| `version.js` | Auto-generated version string consumed by the app at runtime. |
| `package.json` | Project metadata and npm scripts (`start`, `build:offline`, `build:file`). |
| `scripts/proxy.js` | Local dev server + CORS proxy. Auto-enables local proxy for SSL Labs and HTTP header tools. |
| `scripts/build-dev.js` | Inlines all JSX into a single `<script type="text/babel">` for `file://` use. |
| `scripts/bundle.js` | Full offline bundler — fetches CDN assets and inlines everything. |
| `scripts/extract-components.js` | One-time script used to split the original monolithic HTML into `components/`. |
| `scripts/sync-version.js` | Writes the current `package.json` version into `version.js` after `npm version`. |
| `Dockerfile` | Builds a Docker image running `proxy.js`. |
| `docker-compose.yml` | Single-service compose file for the proxy server. |
| `.github/workflows/release.yml` | GitHub Actions CI — builds offline HTML, Docker image, and publishes to GHCR + Docker Hub. |

## Features

- **IPv4**: CIDR subnet calculator, VLSM planner, supernet summarization, range-to-CIDR, split & merge, overlap detection, wildcard masks, ACL generator, DHCP scope planner, IP converter, subnet visual map
- **IPv6**: Subnet calculator, expand/compress, address classification, transition mechanisms (NAT64/DNS64, DS-Lite, 464XLAT, 6in4/GRE)
- **Multicast**: Full suite — planner, IP-MAC mapping, GLOP calculator, solicited-node, collision checker, IPv6 multicast builder, plus a comprehensive reference
- **Switching**: STP/RSTP/MST reference, LACP/port-channel simulator, VXLAN reference, interface config generator
- **Routing**: IP protocol reference, MPLS label breakdown, VPN/IPsec architect, BGP looking glass, ASN lookup
- **Infrastructure**: OSI model, packet header map, QoS/DSCP, MTU & encapsulation calculator, WLAN planner, WiFi QR code generator
- **Media**: IPFM / NBM / PTP reference for SMPTE ST 2110 broadcast environments
- **Diagnostics**: DNS lookup, SSL/TLS inspector, HTTP header analyzer, remote ping/MTR, IP geolocation, Wireshark toolkit, ping sweep planner, SysTool CLI builder (Linux + Windows), bandwidth calculator, Cypher Deck (encode/decode/hash/JWT/XOR), CLI quick reference, self-signed cert generator, port reference
- **Education**: Network Arcade — Packet Rain, Subnet Sprint, and IPv6 Gauntlet interactive games
- All computation runs in the browser — no server needed

See [TOOLS.md](TOOLS.md) for a detailed feature guide.

## Development Workflow

### Mode comparison

| Mode | How to run | Internet needed | Build step | CORS proxy | Best for |
|------|-----------|-----------------|------------|------------|----------|
| **HTTP dev** | `python3 -m http.server 8081` | Yes (CDN + fonts) | None | No (edit & refresh) | Active development |
| **file:// dev** | Open `rda-net-kit-file.html` directly | Yes (CDN + fonts) | `node scripts/build-dev.js` after each change | No | Testing without a local server |
| **Proxy server** | `node scripts/proxy.js` | Yes (CDN + fonts) | None | **Yes** — local proxy auto-enabled | Full tool testing (SSL Labs, HTTP headers) |
| **Docker** | `docker run -p 8080:8080 rda-net-kit` | Yes (CDN + fonts) | `docker build` | **Yes** — local proxy auto-enabled | Sharing / deployment |
| **Full offline** | `node scripts/bundle.js` then open output | Only during build | Yes | No | Distribution / air-gapped use |

**Why file:// dev needs a build step:** Babel Standalone fetches `src=` script files via XHR, which modern browsers block on the `file://` protocol. `build-dev.js` inlines all JSX components into a single `<script type="text/babel">` block so no XHR is needed.

**Why some tools need a CORS proxy:** APIs like [SSL Labs](https://api.ssllabs.com) don't send `Access-Control-Allow-Origin` headers — they're designed for server-to-server calls. The proxy server and Docker image include a local CORS proxy at `/proxy/fetch?url=` that the app detects and uses automatically. GeoIP tools (ipwhois.app, ip-api.com, ipinfo.io) work without a proxy because they explicitly support browser CORS.

### Making changes

Components live in `components/` as individual `.jsx` files. The main HTML (`rda-net-kit.html`) loads them via `<script type="text/babel" src="...">` tags.

The IP utility functions and OUI database are loaded via `<script src>` tags:
- `ip-utils.js` — standalone module (exports to `window.IPv4` / `window.IPv6`)
- `oui-db.js` — MAC vendor lookup table (exports to `window.OUI_DB`)

Both are automatically inlined by the bundler when building the offline version.

### Dev via HTTP (no build step)

Serve the project root over HTTP so Babel can fetch the component files:

```bash
python3 -m http.server 8080
# then open http://localhost:8080/rda-net-kit.html
```

Edit any file in `components/` and refresh the browser.

### Dev with proxy server (recommended for tool testing)

Runs a local Node.js server that serves the app and provides a CORS proxy for APIs like SSL Labs. No build step needed — edit a component and refresh.

```bash
node scripts/proxy.js
# then open http://localhost:8080
```

Tools that need a CORS proxy (SSL Inspector, HTTP Header Analyzer) will auto-detect the local proxy via `window.LOCAL_PROXY`. Custom port:

```bash
PORT=3000 node scripts/proxy.js
```

### Dev via file:// (build step required)

Modern browsers block Babel Standalone's XHR when opening HTML from the filesystem directly. Run the dev build to inline all components:

```bash
node scripts/build-dev.js
# then open rda-net-kit-file.html in any browser
```

Re-run after editing any component file. A custom output path is also supported:

```bash
node scripts/build-dev.js -o ~/Desktop/nettools.html
```

### Updating the OUI database

The MAC address vendor database comes from [maclookup.app](https://maclookup.app/downloads/json-database) and is free to use with attribution.

To refresh it:

```bash
curl -sL 'https://maclookup.app/downloads/json-database/get-db' -o /tmp/maclookup.json
python3 -c "
import json
data = json.load(open('/tmp/maclookup.json'))
oui = {e['macPrefix'].replace(':','').upper(): e['vendorName'] for e in data if len(e['macPrefix'].replace(':','')) == 6}
js = '// MAC OUI Database — sourced from maclookup.app\n// License: Free to use with attribution. See https://maclookup.app/terms-and-conditions\n// Database: https://maclookup.app/downloads/json-database\n// ' + str(len(oui)) + ' entries\nconst OUI_DB = ' + json.dumps(oui, separators=(',',':')) + ';\nwindow.OUI_DB = OUI_DB;\n'
open('oui-db.js','w').write(js)
print(f'Done: {len(oui)} entries')
"
```

### Docker

Builds a self-contained image with the proxy server included. No npm install needed.

```bash
docker build -t rda-net-kit .
docker run -p 8080:8080 rda-net-kit
# then open http://localhost:8080
```

The image serves `rda-net-kit.html` as the root and the CORS proxy at `/proxy/fetch?url=`. React, Babel, and fonts are still loaded from CDN at runtime — users need internet access.

Or use Docker Compose:

```bash
docker compose up
```

### Building the offline version

Requires Node.js (no npm install needed — zero dependencies beyond Node built-ins).

```bash
node scripts/bundle.js rda-net-kit.html
```

This creates `rda-net-kit-offline.html` in the same directory.

To specify a custom output path:

```bash
node scripts/bundle.js rda-net-kit.html -o my-output.html
```

#### What the bundler does

1. Finds all `<script src="...">` tags — both remote (CDN) and local (e.g. `ip-utils.js`, `oui-db.js`)
2. Fetches remote scripts, reads local scripts from disk
3. Finds external CSS `<link rel="stylesheet">` tags and fetches them
4. Parses CSS to find all font/image `url()` references and fetches those
5. Gzip-compresses text resources, base64-encodes everything into a manifest
6. Generates a single self-extracting HTML file that decodes all assets at load time

#### When to rebuild

Re-run `scripts/bundle.js` whenever you change:
- Any component in `components/` or the HTML in `rda-net-kit.html`
- The `ip-utils.js` logic
- CDN library versions (e.g., upgrading React)

Font and CDN assets are fetched live during bundling, so the build always gets the latest version from the URLs in the source HTML.

## Release Workflow

### CI/CD (GitHub Actions)

When a `v*.*.*` tag is pushed, `.github/workflows/release.yml` automatically:
1. Builds the offline HTML (`RdaNetKit-vX.Y.Z-Offline.html`) and uploads it as a GitHub Release artifact
2. Builds and pushes a Docker image to GHCR and Docker Hub with semver tags
